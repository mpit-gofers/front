Контекст приложения:
- Это сервис, где пользователь задаёт вопрос на естественном языке.
- Система интерпретирует запрос, генерирует SQL, возвращает результат в виде таблицы и графика.
- Также есть библиотека отчётов и просмотр сохранённого отчёта.
- Основные маршруты:
  - "/" — главный экран
  - "/loading/:queryId" — экран обработки запроса
  - "/result/:queryId" — экран результата
  - "/reports" — библиотека отчётов
  - "/report/:reportId" — экран просмотра отчёта
- Сейчас это MVP. Базовая логика уже есть. Нужно в первую очередь улучшить интерфейс, навигацию, структуру экранов, визуальную иерархию, читаемость, состояния и UX.

Что важно:
- Не превращай интерфейс в перегруженный BI-инструмент.
- Это интерфейс для non-tech пользователя: менеджер, маркетолог, операционный сотрудник.
- Интерфейс должен быть простым, современным, “умным”, чистым, визуально уверенным.
- Нужен стиль уровня современного AI SaaS продукта.
- Сохрани ощущение, что это AI-ассистент для аналитики, но без чрезмерной “чатовости”.
- Удели особое внимание spacing, visual hierarchy, alignment, readability, empty states, loading states, CTA, navigation.

Главная задача этапа:
Сделай редизайн текущего MVP так, чтобы он выглядел как аккуратный, цельный продукт, готовый к демо на хакатоне.

Сделай следующие улучшения:

1. Общий стиль
- Сделай единый визуальный стиль для всех экранов.
- Используй современный минималистичный SaaS UI.
- Светлая тема, чистый фон, хорошие отступы, большие контейнеры, аккуратные карточки.
- Хорошая типографика: чёткая иерархия заголовков, подзаголовков, подписей и основного текста.
- Сделай интерфейс визуально “дороже”, но без перегруза декоративностью.
- Не используй слишком яркие кислотные цвета.
- Добавь мягкие тени, радиусы, чистые разделители.
- Сохрани ощущение аналитического продукта, а не лендинга.

2. Root layout / глобальная структура
- Улучши общий layout приложения.
- Сделай аккуратный top navigation / header.
- В шапке оставь:
  - логотип / название продукта
  - навигацию: New Query / Reports
- Добавь ощущение приложения, а не набора отдельных страниц.
- Сделай ширину контента удобной для аналитических экранов.
- Добавь консистентные паддинги и max-width.
- Навигация должна быть понятной и аккуратной.

3. Главный экран "/"
Нужно переработать Home screen:
- В центре внимания должен быть основной input для текстового запроса.
- Сделай hero-like рабочую область, но без лендинговой избыточности.
- Добавь:
  - заголовок, который объясняет ценность продукта
  - краткий supporting text
  - большой текстовый input / textarea
  - primary CTA
  - блок примеров запросов
- Примеры запросов должны быть кликабельными “chips” или карточками.
- Интерфейс должен подталкивать пользователя сразу попробовать запрос.
- Сделай композицию так, чтобы пользователь сразу понял основной сценарий.
- Добавь аккуратные empty/help states.
- Сделай screen очень чистым и сильным визуально.

4. Loading screen "/loading/:queryId"
Нужно полностью улучшить UX loading screen:
- Сейчас это просто промежуточный экран, но его нужно превратить в “умный” этап обработки.
- Покажи пользователю, что система:
  - анализирует запрос
  - определяет метрику
  - подбирает SQL
  - готовит визуализацию
- Сделай loading screen полезным, а не пустым.
- Можно использовать step-by-step progress card.
- Состояние должно создавать ощущение работы AI.
- Добавь мягкую анимационную логику или прогресс-индикацию.
- Не делай слишком мультяшно.
- Должно выглядеть профессионально и технологично.

5. Result screen "/result/:queryId"
На этом этапе не нужно делать сложный enterprise dashboard.
Нужно просто привести экран результата к хорошему UX:
- Чёткий header результата
- Основной инсайт/summary в верхней части
- Дальше таблица и график
- Также explain и SQL должны быть доступны, но не должны мешать восприятию результата
- Выстрой хорошую иерархию:
  1. что получилось
  2. визуализация
  3. таблица
  4. объяснение / SQL
- Сделай action bar:
  - Save report
  - Edit query / Back
- Убери визуальный шум
- Сделай таблицу читаемой
- Сделай график аккуратным и современным
- Explain и SQL лучше оформить в collapsible sections / cards / tabs / expandable blocks, чтобы они не перегружали экран
- Не прячь их полностью, но снизь визуальный приоритет относительно результата

6. Reports library "/reports"
Сейчас это библиотека отчётов. Улучши UX:
- Сделай экран библиотекой, а не просто списком
- Добавь:
  - заголовок
  - описание
  - поиск
  - фильтры или хотя бы визуальное разделение
- Отчёты показывай в виде аккуратных карточек или чистого списка-каталога
- На карточке отчёта покажи:
  - название
  - краткое описание / query preview
  - дату / последнее обновление
  - CTA для открытия
- Если отчётов нет — сделай красивый empty state
- Экран должен визуально поддерживать идею “переиспользуемой аналитики”

7. Report view "/report/:reportId"
Нужно улучшить экран просмотра отчёта:
- Добавь хороший header отчёта
- Покажи:
  - название
  - описание
  - связанный запрос
  - дата создания / обновления
- Основной результат должен быть показан как полноценный saved report
- Кнопки:
  - rerun report
  - back to reports
- Сделай экран более “report-like”, чем обычный result page
- Но сохрани общий визуальный стиль с экраном результата

8. Состояния интерфейса
Очень важно улучшить все состояния:
- loading
- empty
- no data
- error
- validation message
- reports empty state
- missing report state
Все эти состояния должны быть:
- аккуратными
- дружелюбными
- не техническими
- визуально консистентными

9. UX детали
- Улучши читаемость длинных запросов и SQL.
- Добавь правильные отступы между секциями.
- Сделай понятные CTA.
- Добавь hover/focus/active states.
- Продумай mobile-ish responsiveness, но приоритет desktop-first.
- Не ломай основной flow.
- Не меняй концепцию продукта.
- Не усложняй сценарии.
- Улучши именно существующий MVP.

10. Важные ограничения
- Не добавляй авторизацию.
- Не добавляй сложные BI-фильтры.
- Не добавляй дашборд-билдер.
- Не превращай главный экран в чат.
- Не добавляй лишние сущности, которых нет в MVP.
- Делай реалистичный UI для хакатонного продукта, который может впечатлить жюри.

11. Что хочу получить
Сгенерируй улучшенный UI/UX для всего приложения:
- Home
- Loading
- Result
- Reports Library
- Report View
- Empty/Error states
Сохрани логическую структуру текущего приложения, но сделай интерфейс цельным, современным и убедительным.

Приоритет:
1. Понятность для non-tech пользователя
2. Визуальная чистота
3. Хорошая иерархия результата
4. Убедительность демо
5. Цельный SaaS-стиль
